# 机械工业时代 Factory

一个基于原生 HTML/CSS/JavaScript 开发的工业网络建造游戏原型。玩家通过矿源、采矿机、熔炉、铸造厂、组装工厂、仓库、发电机、电杆和封装分组节点搭建资源生产网络，完成第一阶段工业目标。

## 运行方式

当前版本已经内置服务器能力：玩家可以通过网页注册、登录，并把多个存档保存到服务器数据库中。

本地开发运行：

```bash
npm start
```

打开浏览器访问：

```text
http://localhost:5174
```

服务器默认监听 `0.0.0.0:5174`，同一局域网或云服务器放行端口后，其他玩家可以通过服务器 IP 或域名访问。

> 服务器需要 Node.js 24 或更高版本，因为服务端使用了 Node 内置 SQLite。

## 一键部署

### Docker 部署（推荐）

Linux / 云服务器：

```bash
git clone https://github.com/Ezreal0011/Factory.git
cd Factory
chmod +x deploy.sh
./deploy.sh
```

Windows / PowerShell：

```powershell
git clone https://github.com/Ezreal0011/Factory.git
cd Factory
.\deploy.ps1
```

部署完成后访问：

```text
http://服务器IP:5174
```

如需修改端口：

```bash
PORT=8080 ./deploy.sh
```

Windows：

```powershell
.\deploy.ps1 -Port 8080
```

### 手动 Docker Compose

```bash
cp .env.example .env
# 修改 .env 里的 SESSION_SECRET
docker compose up -d --build
```

数据库会保存在：

```text
./data/factory.sqlite
```

这个目录已经通过 Docker volume 挂载，升级镜像或重启容器不会丢失玩家账号和存档。

## 服务器功能

- 玩家注册
- 玩家登录
- Cookie 会话保持
- 管理员 GM 权限校验
- 多存档创建
- 多存档读取
- 覆盖存档
- 删除存档
- 存档版本迁移
- 静态托管游戏前端
- SQLite 持久化账号和存档数据

主要 API：

```text
POST /api/auth/register
POST /api/auth/login
POST /api/auth/logout
GET  /api/me
GET  /api/saves
POST /api/saves
GET  /api/saves/:id
PUT  /api/saves/:id
DELETE /api/saves/:id
GET  /api/admin/gm-session
```

## 环境变量

复制 `.env.example` 为 `.env`：

```text
PORT=5174
HOST=0.0.0.0
DATA_DIR=./data
SESSION_SECRET=change-this-to-a-long-random-secret
COOKIE_SECURE=false
GAME_VERSION=0.1.0
ADMIN_USERNAME=
ADMIN_PASSWORD=
BACKUP_DIR=./backups
BACKUP_KEEP=30
```

正式部署建议：

- `SESSION_SECRET` 改成足够长的随机字符串。
- 如需开启 GM 面板，设置 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD`，服务器启动时会自动创建或更新管理员账号。
- 使用域名和 HTTPS 后，把 `COOKIE_SECURE=true`。
- 云服务器安全组放行 `PORT` 对应端口。

## GM 管理

当前 GM 面板已经从前端固定密码改为服务端管理员校验：

1. 在 `.env` 中设置 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD`。
2. 重启服务器。
3. 用该管理员账号登录游戏。
4. 点击标题旁的 `GM管理` 按钮即可打开 GM 面板。

普通玩家点击 GM 面板会被服务器拒绝。后续接入正式服务端时，可以继续把这里扩展成角色权限、操作日志和后台审计。

## 当前核心玩法

- 放置矿源、采矿机、熔炉、高温熔炼炉、铸造厂、组装工厂、仓库、发电机、电杆和封装节点。
- 拖拽物流线连接设备输出口和输入口。
- 拖拽电力线连接发电机、电杆和用电设备。
- 矿源拥有有限储量，当前上限为 `50000`，并有 3 个输出口。
- 采矿机从矿源开采对应矿物，再输出到生产设备。
- 熔炉、高温熔炼炉、铸造厂和组装工厂支持配方选择。
- 配方选择使用分组弹窗，展示材料、产物和用途说明。
- 仓库支持单资源锁定、清空后自动解除锁定、输入/输出口开关和出口限流。
- 支持服务器多存档创建、覆盖、读取和删除。

## 封装分组系统

- 可多选节点后创建封装分组。
- 双击封装分组节点可进入组内画布。
- 组内可使用三类封装节点声明对外接口：
  - `封装输入`：组外输入 1/2/3 对应组内输出 1/2/3。
  - `封装输出`：组内输入 1/2/3 对应组外输出 1/2/3。
  - `封装电力`：组外电力进入组内电网。
- 封装输入和封装输出使用端口独立缓存，不共享真实物流判断。
- 每个封装端口缓存上限为 `1`，接口节点只作为过渡接口，不作为隐藏仓库。
- 当端口换线、换资源、删除连线、删除节点、清空连线或读档时，会自动整理端口绑定。
- 如果新资源和旧端口锁定不一致，会自动清理该端口旧缓存和旧物流包，再重新锁定新资源。

## 封装节点显示

- 封装分组节点使用专属接口面板卡，和普通生产节点区分显示。
- 卡片宽度更大，左侧显示 `IN 1/2/3`，右侧显示 `OUT 1/2/3`。
- 每个端口显示端口编号、资源色点和资源名称。
- 未绑定端口显示 `待绑定`，便于排查封装接口是否声明完整。
- 电力接口显示 `PWR 接入` 或 `PWR 未接入`。
- 远景模式显示接口摘要，例如 `IN 2 / OUT 3 / PWR`。
- 中景模式压缩资源名，近景模式显示完整端口详情。
- 选中封装节点时会展开更完整的信息。

## 节点状态显示

- 普通节点标题只显示节点名称，避免状态原因挤压标题换行。
- 暂停原因、电力状态等信息以 TAG 形式显示在节点卡内容区。
- 近景显示完整 TAG，中景保留核心 TAG，远景隐藏文字 TAG。
- 选中节点时，无论缩放级别都会显示状态 TAG，方便排查问题。

## 电力系统

- 发电机消耗煤矿产生电力。
- 电杆可作为电力中继，并拥有 3 个输出口。
- 电力线有容量限制。
- 电力线负载会递归计算下游需求。
- 电杆输入线负载 = 电杆自身耗电 + 下游所有输出分支耗电。
- 电网监控使用动态图形仪表：
  - `< 60%`：绿色 / 稳定
  - `>= 60%`：黄色 / 高负载
  - `>= 80%`：红色 / 过载
  - `>= 100%` 或真实过载：红色 / 已超载
- 过载电力线会闪烁，高负载会在监控和警报中提示。

## 物流系统

- 物流包沿连线路径移动，移动速度不随线路长度变化。
- 物流线使用折线路径，避免曲线造成视觉混乱。
- 物流监控使用 5 秒滑动平均吞吐，避免瞬时数据跳动。
- 物流线健康状态：
  - 正常：蓝色
  - 高负载：黄色
  - 堵塞：红色闪烁
- 支持显示物流线、显示电力线、显示所有线三种视图。

## 操作说明

- 点击底部设备按钮放置节点。
- 拖拽节点移动位置。
- 拖动画布可平移视图。
- 鼠标滚轮以鼠标所在位置为中心缩放画布。
- 从输出口拖到输入口创建物流线。
- 从黄色电力输出口拖到黄色电力输入口创建电力线。
- 拖线过程中按 `Esc` 取消。
- 点击已有连线选中，按 `Delete` 删除。
- 点击节点后可在右侧查看状态、缓存、配方和容量。
- 多选节点后可整体移动或创建封装分组。

## 阶段目标

游戏包含第一阶段目标链：

1. 建立铁矿采集线
2. 稳定生产铁锭
3. 接入稳定电网
4. 进入钢铁生产
5. 准备核心组件
6. 完成机械核心目标

目标完成后会奖励设备或线缆库存，引导玩家逐步扩展产线。
